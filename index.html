CppDSA is a deep dive into data structures and algorithms using C++ and the STL. This project is dedicated to exploring, benchmarking, and profiling the rich set of containers and algorithms provided by the C++ Standard Library. In addition, you'll find detailed documentation and tests to help you understand both the theoretical and practical aspects of these topics.

Overview

CppDSA is a comprehensive project focused on:

  • STL Data Structures & Algorithms:
    Detailed implementations, usage guides, and performance comparisons of the C++ Standard Library components.
  • Benchmarking:
    Microbenchmarks using Google Benchmark to measure the performance of various container operations and algorithms.
  • Profiling:
    Techniques and examples using gprof, Valgrind (Callgrind), and perf to analyze performance bottlenecks.
  • Documentation:
    In-depth markdown guides and Doxygen-generated documentation to explain the inner workings and design of STL components.
  • Testing:
    A suite of unit tests (using Google Test) to verify the correctness of examples and implementations.

Getting Started

Prerequisites

  • Compiler: A modern C++ compiler with C++20 support (e.g., GCC 10+, Clang 10+, or MSVC 2019+).
  • CMake: Version 3.16 or later.
  • Google Benchmark: Integrated via CMake.
  • Google Test: For running unit tests.
  • Optional Tools:
    • Profiling: gprof, Valgrind (Callgrind), perf.
    • Documentation: Doxygen.

Installation

Clone the repository and navigate to the project root:

git clone https://github.com/MangaD/CppDSA.git
cd CppDSA

Building and Running

Building the Project

The project uses CMake for configuration and building:

  1. Create a build directory and configure:
    mkdir build && cd build
    cmake .. -DCMAKE_BUILD_TYPE=RelWithDebInfo
  2. Build the project:
    make -j$(nproc)

Running Tests

Unit tests are located in the tests/ directory and use Google Test. After building, run:

cd build
ctest --output-on-failure

Alternatively, run the test executable directly:

./tests/unit_tests

Running Benchmarks

Benchmarks are implemented using Google Benchmark. From the build directory, you can run each benchmark executable, for example:

./vector_benchmark
./map_benchmark
./unordered_map_benchmark
./deque_benchmark

Documentation

Detailed documentation is available in the docs/ directory:

  • Markdown Guides:
    For instance, docs/vector.md provides an in-depth look at std::vector.
  • Doxygen:
    A Doxygen configuration file is provided to generate HTML documentation. To generate docs:
    cd docs
    doxygen Doxyfile

Profiling

Profiling examples and guides are integrated into the project documentation. Use tools such as:

  • gprof:
    Compile with -pg to generate profiling data.
  • Valgrind (Callgrind):
    Use to generate detailed call graphs.
  • perf:
    Use for sampling-based profiling on Linux.

Refer to the profiling guide in the docs for more details.

Continuous Integration (CI/CD)

CI/CD pipelines are set up using GitHub Actions. The workflows in .github/workflows/ci.yml include steps to:

  • Build the project.
  • Run unit tests.
  • Execute benchmarks.
  • (Optionally) Archive profiling artifacts.
